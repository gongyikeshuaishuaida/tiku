## 二、文件命名规则

### 2.1 题目文件命名

**格式**：`时间-组织-年级-类型-题号.md`

**示例**：
- `202604-湖衢丽-高三-期中-08.md`
- `202605-义乌柯桥-高二-月考-15.md`

**说明**：
- 时间：6位数字，如 `202604` 表示2026年4月；若原始文件使用 `2605` 这类 4 位短年份，入库时统一展开为 `202605`
- 组织：如 `湖衢丽`、`金华十校`、`浙江` 等
- 年级：如 `高一`、`高二`、`高三`
- 类型：`期中`、`期末`、`月考`、`学考`、`期初`、`一模`、`二模`、`三模`
- 题号：**2位数字**，如 `01`、`15`

**重要**：题号必须统一为**两位数**格式
- ✅ 正确：`202604-湖衢丽-高三-期中-01.md`、`202605-义乌柯桥-高二-月考-15.md`
- ❌ 错误：`202604-湖衢丽-高三-期中-001.md`、`202604-湖衢丽-高三-期中-1.md`

题目文件名必须和 frontmatter 中的 `id` 保持一致，即 `id.md`。

**专题改编/原创题例外**：
- 若专题卷中的改编题或原创题需要独立入库，文件名、frontmatter `id`、正文一级标题统一使用 `改编队列xx` 或 `原创队列xx`，其中 `xx` 为该类题在题库中的两位顺序编号，从 `01` 起递增，不沿用专题卷中的题号，例如 `改编队列01.md`、`原创队列04.md`。
- 改编题 frontmatter 的 `来源` 必须链接到被改编的原题 note，例如 `来源: "[[09队列/202406-宁波九校-高二-期末-15|202406-宁波九校-高二-期末-15]]"`；原创题 `来源` 写 `原创`。
- 这类题的 `试卷` 字段写 `改编出题` 或 `原创出题`，`题号` 保持两位数字字符串。

### 2.2 试卷文件命名

- 试卷统一存放在 `试卷/` 文件夹
- 试卷文件优先保存 PDF 原稿；答案若为 PNG、DOCX 等非 PDF 格式，先转换为 PDF 后再入库
- 命名格式：`时间-组织-年级-类型.pdf`
- 示例：`202604-金华十校-高三-期中.pdf`
- 答案文件统一命名为 `时间-组织-年级-类型-答案.pdf`，如 `202605-诸暨-高三-三模-答案.pdf`
- 同一试卷下的题目 `id` 使用该试卷名作为前缀，例如 `202604-金华十校-高三-期中-01`
- 只有收到或处理 PDF 原稿时才添加试卷文件，不额外创建试卷索引 md 文件
- 历史题目若已使用旧格式 `时间-组织-类型-题号`，未做专项迁移前保持原 `id` 不变；新处理的试卷必须使用带年级格式

## 四、格式规范

### 4.1 标题规范
- ✅ 使用 `# 题目 题号` 作为题目标题，如 `# 题目 01`
- ✅ 专题改编/原创题按命名例外使用 `# 改编队列xx` 或 `# 原创队列xx`
- ❌ 不要使用 `# 题目内容`、`## 问题`、`## 选项` 等标题
- 选项直接作为题目内容的一部分，不单独分节
- 选择题每个题干末尾必须统一写 `（   ）`，不要省略括号，也不要使用 `（▲）`；若原题题干末尾有句号、问号等标点，统一去掉题干末尾标点后保留 `（   ）`

### 4.2 Frontmatter 标签
- `tags` 只保留状态类和知识点类标签，例如 `状态/未练习`、`知识点/03python基础`
- 不写可由字段推导的冗余标签，例如 `题库/信息技术`、`题型/选择题`

### 4.3 代码格式
- **代码块**：使用 ` ```python ... ``` ` 格式
- 连续 Python 程序段必须放入 fenced code block，不能裸写在正文中；即使原卷排版没有框，也要用 ` ```python ` 保证 Markdown 渲染正确
- **行内代码**：Python表达式用反引号包裹，如 `` `df["列名"]` ``、`` `a[i]` ``
- 特别注意：包含 `==`、`[]`、`()` 的表达式需要用反引号
- 选择题选项中的短代码或表达式可以保留为选项文本，必要时只用行内反引号，不强行改成代码块

### 4.4 题目排版质量
- 不允许把 OCR 原始输出整段粘贴为题干；必须按原卷重排为可阅读的 Markdown
- 选择题必须拆成题干和 `- A.`、`- B.`、`- C.`、`- D.` 四个独立选项；不要写成裸 `A.`、`B.`、`C.`、`D.`。网页展示时如不需要项目符号，由前端 CSS 隐藏列表样式
- 材料选择题每个 md 保留完整公共材料，但只保留当前题号的一道选择题
- 非选择题必须按小问编号排版，例如 `(1)`、`(2)`、`①`、`②`，程序填空必须保留代码上下文和空位
- 表格题优先还原为 Markdown 表格；无法可靠还原时，必须裁切原表格图片并插入对应位置
- 流程图、树、栈/队列/链表结构图、统计图、界面截图、数据表截图等不能只保留 OCR 文字，必须裁切图片并插入题干对应位置
- 若 OCR 不能可靠识别题干、代码、选项或答案，不得写成正式题目；只能标记为草稿/待复核，并在 `题目核验.md` 中说明问题
- 结构校验通过只表示 frontmatter 和基础字段正确，不代表题干排版、图片裁切和答案质量合格

### 4.5 图片引用
- 图片存放在 `attachments/` 文件夹
- 引用格式：`![[attachments/图片名.png]]`
- 图片命名：`试卷时间试卷简称_题号_序号.png`，如 `202604湖衢丽_p02_img01.png`
- 图片应插入到题干引用该图的恰当位置，例如“如第14题图a所示”“如下图所示”“流程图如图所示”之后；不允许把所有图片统一堆到 `---` 或 `## 答案` 前
- 如果图片本身已经包含“第14题图a”“第14题图b”等图注，Markdown 正文不要再额外写独立图注行
- 多张图片按题干引用顺序分别放在对应说明或小问附近

### 4.6 图片核验
- 每个含图片的题目必须填写 frontmatter 的 `图片核验` 字段
- 可选值：`无图片`、`待核验`、`已通过`
- `已通过` 表示图片确认为题目对应的局部截图，并且插入位置与题干引用位置一致，例如统计图、流程图、树结构或选项图
- 不允许把整页试卷截图作为题目图片保存
- PDF 提取后，凡是自动截取的图片先标记为 `待核验`，人工确认后再改为 `已通过`
- 图片下方不要添加 `> 上图：...` 这类说明文字
- 如果图片位置错误、集中堆在正文末尾，或正文重复写出图片内部已有图注，应保持或改为 `待核验`，直到人工确认
- 每次处理完一批新试卷后，必须更新 `图片核验.md`：先清空上一批图片，再按试卷和题号列出本批新生成或更新题目的图片引用，格式使用 `![[attachments/图片名.png]]`，并附上题目 note 链接和 `图片核验` 状态

### 4.7 题目核验
- 每次处理完一批新试卷后，必须更新 `题目核验.md`
- 更新时先清空上一批题目链接，再按试卷分组列出本批新生成或更新的题目 md
- 每行至少包含题号、题型、主知识点和题目 note 链接，便于逐题检查题干、答案、分类和来源是否正确
- `题目核验.md` 使用 Markdown 表格时，题目链接必须写成不带别名的 Obsidian 链接，例如 `[[03python基础/202606-九校-高二-期末-14]]`；禁止在表格单元格中写 `[[路径|别名]]`，因为 `|` 会被 Markdown 表格识别为列分隔符，导致表格多列或错列
- 更新 `题目核验.md` 后必须检查表头和每一行都是 4 列：`题号`、`题型`、`知识点`、`题目 md`

### 4.8 答案和解析
- 每道题必须包含 `## 答案` 和 `## 解析`
- 答案和解析由提取者在生成题目时完成，不留空，不写 `待补充`
- 选择题答案格式：`**正确答案：** A`
- 多选题答案格式：`**正确答案：** A、C`
- 填空题答案格式：逐空列出，例如 `① ...；② ...`
- 解析要说明关键判断依据、排除理由或代码执行过程
- 解析不需要写成长篇讲义，但必须足够支持复习时理解为什么这样作答

## 五、知识点分类规则

### 5.1 分类依据
根据题目涉及的主要知识点放入对应文件夹

### 5.2 分类优先级
- `02算法` 只放算法概念题和流程图题，例如算法三要素、算法五个特征、流程图执行。只要是 Python 代码题，即使涉及贪心、滑动窗口、状态模拟等算法思想，也不要放入 `02算法`。
- 试卷第 `13`、`14` 题中的一般 Python 代码题优先放入 `03python基础`。如果涉及数组、排序等内容，在 frontmatter 的 `知识点` 中补充 `08数组`、`13查找与排序` 等次级知识点。
- `04数据处理` 只放明确使用 pandas、Excel 数据表、DataFrame、`read_excel`、`groupby`、统计图表绘制等数据表处理的题。普通列表、数组或传感器数据模拟处理不放入 `04数据处理`。
- 试卷第 `15` 题一般按数据结构归入 `08数组`、`09队列`、`10栈`、`11链表`。优先判断是否属于队列、栈、链表；若都不属于，再放入 `08数组`。若同时涉及排序，在 frontmatter 的 `知识点` 中补充 `13查找与排序`。
- 出现 `head`、`p`、`q`、节点指针、后继下标、`-1` 作为空指针等链表模拟特征时，主分类归入 `11链表`，即使底层数据结构用 Python 列表或数组存储
- 综合题同时涉及 `06信息系统` 和 `04数据处理` 时，按小问数量和主要考查内容决定主目录；frontmatter 的 `知识点` 中同时保留 `06信息系统` 和 `04数据处理`，保证两类检索都能命中。
- 加密或解密相关题补充细分知识点 `加密解密`，常见于 `03python基础`、`07信息安全`、`08数组`。
- 随机数相关题补充细分知识点 `随机`，常见于 `03python基础`、`08数组`，例如使用 `import random`、`randint`、随机生成数据或判断随机程序可能结果的题。
- 使用累计量、区间和或连续区间统计量快速查询思想的代码题补充 `前缀和`，常见于 `03python基础`、`07信息安全`、`08数组`、`09队列`、`10栈`、`11链表`。
- `09队列` 中涉及固定或可变窗口维护连续区间的题补充 `滑动窗口`；涉及队首队尾回绕、取模下标、判空判满的题补充 `循环队列`。
- `10栈` 中通过栈维护单调递增或递减关系、求最近更大/更小元素或区间贡献的题补充 `单调栈`。
- `11链表` 中涉及按位置、顺序或规则修改后继指针插入节点的题补充 `链表插入`。
- 以上细分知识点只作为辅助检索标签，不改变主目录归类。

### 5.3 跨知识点题目
- 非材料题涉及多个知识点时，主文件只放入一个最主要的知识点文件夹
- 其他相关知识点写入 frontmatter 的 `知识点` 字段
- 例如：一道题主要考查栈，同时涉及队列，则文件放入 `10栈`，并在 `知识点` 中同时写入 `10栈`、`09队列`

### 5.4 知识点标签
在 frontmatter 的 `知识点` 字段中：
- 第一项写大知识点（如 `01数据与编码`）
- 后续可写细分知识点
- 其他知识点文件夹下的题目若涉及已设定的细分知识点，应优先复用现有标签；只有现有标签都不符合题目核心考查点时，才新建新的细分知识点
- 若新增细分知识点，应在 `tags` 中同步写入 `知识点/<细分知识点>`，例如 `知识点/加密解密`、`知识点/随机`、`知识点/前缀和`、`知识点/滑动窗口`、`知识点/循环队列`、`知识点/单调栈`、`知识点/链表插入`

### 5.5 题目唯一 ID
- 每道题必须填写稳定的 `id`
- 格式：`YYYYMM-组织-年级-类型-题号`
- 示例：`202604-金华十校-高三-期中-01`
- 题目文件名必须与 `id` 一致，即 `202604-金华十校-高三-期中-01.md`
- 题目移动到其他知识点文件夹时，`id` 和文件名都不变

### 5.6 复习字段
- `完成次数`：累计练习次数，初始为 `0`
- `正确率`：当前统计正确率，使用 0~1 的小数记录，例如 `1`、`0.92`、`0.75`；未练习时留空
- `状态`：可选 `未练习`、`已完成`、`需复习`、`错题`
- `错题原因`：记录错误类型或原因，例如 `概念混淆`、`代码跟踪错误`、`审题错误`
- 批量导入练习统计时，选择题可按外部统计表同步 `完成次数`、`正确率` 和 `状态`

## 六、试卷提取规则

### 6.1 提取范围
- ✅ **只提取信息技术部分**（通常是试卷前半部分）
- ❌ 不提取通用技术部分

### 6.2 提取要求
- 题目内容**原封不动**，不做概括或改写
- 材料选择题要整体提取公共材料，并按题号复制为多个 md 文件
- 例如：`阅读材料，完成选择题1-3`，需要复制 3 份，`id`、文件名和标题题号分别为 `01`、`02`、`03`
- 每份 md 正文只包含“公共材料 + 当前题号的一道选择题”，不得把同一材料下的其它小题题干、选项、答案或解析复制进来
- 原卷中的“回答第 X 至 Y 题”或同义表述，在单题 md 中改写为当前题号，例如 `阅读下列材料，回答第 04 题。`
- 同一题组的公共材料复制到每个相关 md；只服务于某一道小题的题图或代码只放入该题 md
- 材料选择题的 `## 相关链接` 可以列出同一公共材料下的其它选择题 note，便于同题组互跳；这里只放 note 链接，不复制其它小题的题干、选项、答案或解析
- 非材料题或普通题目的 `## 相关链接` 可以列出类似知识点题，优先选择 frontmatter `知识点` 中共享同一细分知识点的题；没有细分知识点时，可选择同一大知识点下考法接近的题
- `类似知识点题` 只放 note 链接，数量控制在 3-5 个，避免把整批同目录题全部堆入；不得复制其它题题干、选项、答案或解析
- 复制后的每份文件按对应题目的主要知识点分类，放入对应文件夹
- 保留原始题干、代码、选项
- 删除 PDF/OCR 残留噪音，例如 `第6 页共6 页`、学科大题页眉页脚、页码行等，不作为题目正文保留
- 提取连续 Python 程序段时必须用 ` ```python ` 代码块包裹，避免缩进、下划线和 Markdown 语法混淆
- 图片需要提取试卷中对应的内容并保存，注意一定要是对应内容，不要整张页面截图进来
- 例如：题干中有统计图，你就只截取统计图进来，题干中有树，你就只截取树进来
- 插入图片前先判断图片服务于哪一句题干或哪一个小问，再放到对应位置；图片内已有图注时，不要在 Markdown 中重复写独立图注

### 6.3 提取后处理
1. 按知识点分类存放到对应文件夹
2. 材料题按题号复制多份，并分别分类
3. 跨知识点题目补充多个 `知识点` 字段
4. 为每道题写出答案和解析
5. 将 PDF 原稿放入 `试卷/`
6. 更新 `图片核验.md`，只展示本批处理产生或更新的题目图片
7. 更新 `题目核验.md`，按试卷展示本批处理产生或更新的题目 md
8. 更新题库索引
